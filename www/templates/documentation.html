Swagger

Il existe déjà une documentation sur les requêtes et les réponses associées. Pour cela il faut se rendre sur la page du swagger.

Le swagger ne détaillant pas les valeurs, cela va être fait ici, pour celles qui peuvent préter à confusion.

Le diagramme de la base de donnée est disponible sur le github et ICI.

Fastmove :

Il s'agit des attaques rapides. Il n'y a pas de temps de recharge, mais simplement une durée du mouvement.

  • id : l'identifiant de l'attaque dans la base de donnée.
  • damage : la puissance de l'attaque, servant à calculer les dégâts.
  • dps : les dégâts par seconde, permet de savoir si une attaque plus faible mais plus rapide est plus efficace.
  • energy : nombre de points de recharge de l'attaque chargée (100).
  • eps : energie par seconde.
  • duration : la durée du mouvement.
  • sound : l'effet sonore de l'attaque.
  • name : le nom de l'attaque.
  • type : le type de l'attaque.

Mainmove :

Il s'agit de l'attaque chargée. Il faut que la barre d'énergie soit suffisament remplie pour pouvoir lancer l'attaque.

  • id : l'identifiant de l'attaque dans la base de donnée.
  • damage : la puissance de l'attaque, servant à calculer les dégâts.
  • dps : les dégâts par seconde, permet de savoir si une attaque plus faible mais plus rapide est plus efficace.
  • energy : le coût en énergie de l'attaque.
  • duration : la durée du mouvement.
  • slot : le nombre d'attaque disponible quand la barre d'énergie est à 100.
  • sound : l'effet sonore de l'attaque.
  • name : le nom de l'attaque.
  • type : le type de l'attaque.

Stats_classics :

Ils renseignent les statistiques de bases issus des précédents jeux pokémon (autre que pokémon Go).

  • attack : coefficient pour calculer les dégâts de l'attaque.
  • attack_spe : coefficient pour calculer les dégâts de l'attaque spéciale.
  • base_expérience : l'expérience minimum du pokémon.
  • base_happiness : le niveau de joie du pokémon lors de son obtention.
  • defense : coefficient pour calculer les dégâts reçu.
  • defense_spe : coefficient pour calculer les dégâts reçu lors d'une attaque spéciale.
  • hp : les points de vie minimum du pokémon.
  • speed : la vitesse minimale du pokémon

Stats_go :

Renseigne les statistiques propres à Pokémon Go.

  • attack : coefficient pour calculer la puissance des attaques du pokémon.
  • defense : coefficient pour calculer les dégâts reçu.
  • hp : les points de vie minimum du pokémon.
  • pc : les points de combat maximum du pokémon.
  • stamina : les points d'endurance du pokémon.
  • buddy_walk : la distance à parcourir avec le pokémon pour obtenir un bonbon.
  • capture_rate : le taux de capture du pokémon.
  • escapre_rate : le taux de fuite du pokémon.

Stats_common :

Ils renseignent les statistiques communs aux différentes versions des jeux Pokémon.

  • height : la taille exprimée en mètre.
  • weight : le poids en Kg.

Pokemon :

Renseigne les statistiques propres à Pokémon Go.

  • id : l'identifiant du pokémon sur le pokédex national.
  • image : lien vers l'image du pokémon.
  • order : l'ordre d'affichage respectant les évolutions des pokémons.
  • scream : lien vers le cri du pokémon.
  • specie : indique l'espèce du pokémon.

Type :

Renseigne les types du pokémon.

  • id : l'identifiant du type.
  • img : lien vers l'image du type.
  • name : le nom du type.

Evolve :

Renseigne les évolutions possibles du pokémon.

  • id : l'identifiant de la chaine d'évolution.
  • level : le stade de l'évolution.
  • from : l'identifiant du pokémon avant l'évolution.
  • to : l'identifiant du pokémon après l'évolution.

Abilitie :

Renseigne les talents du pokémon. Attention, certains pokémon n'ont pas de talents !

  • id : l'identifiant du talent.
  • generation : la génération du talent.
  • description : la description du talent.
  • name : le nom du talent.

Pokedex :

Renseigne la génération (pokedex) du pokémon.

  • id : l'identifiant du pokédex.
  • name : le nom du pokédex.
